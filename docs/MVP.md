## 4. 선정 아이디어 상세 (현행)

### 프로젝트명
**도시 기후 설계자**

### 한 줄 설명
기상청 데이터로 도시를 설계하고, 열섬·대기·재난을 검증하는 AI 웹 체험.

### 배경 — 왜 이 이름인가
**도시 기후 설계자(Urban Climate Planner/Designer)** 는 기후변화 시대에 부상하는 신종 전문 직무다. 폭염·집중호우·해수면 상승 등에 도시가 회복력을 갖도록, 기상·기후 데이터를 바탕으로 **그린·블루 인프라**, **재난 회복력**, 통풍·바람길 등을 고려해 공간을 설계한다.

본 서비스는 그 직무를 **대체하는 전문 CAD가 아니라**, 사용자가 **「도시 기후 설계자」 역할을 체험**하는 교육용 웹 시뮬레이터다.

### 학습 대상
중·고등학생, 기후·재난·환경 교육 수강생 및 관심 있는 일반인 (교사 시연·수업 활용 가능). 초등 단일 개념·퀴즈형이 아닌 **설계 실험** 난이도.

### 사용자의 핵심 행동 (User Action)
타일로 도시를 설계 → 렌즈(열섬·대기·재난·농어업)로 검증 → AI 코치 평가 → 재설계.  
열섬에서 **입문**하고, 레벨이 올라가며 대기·재난·농어업 렌즈를 **단계적으로** 확장한다.

### 해결하려는 문제
기후·기상·대기·재난을 "설명"이 아니라 **직접 설계 실험**으로 체득하게 한다.  
여러 환경 문제를 동시에 견디는 **균형 잡힌 도시 설계**를 학습한다.

### 체험 ↔ 실제 직무 (교육용 근사)

| 실제 직무 | 본 체험 |
|----------|---------|
| 그린·블루 인프라 | 공원·가로수·수변·습지 타일 |
| 재난 회복력 | 방조제·배수로·대피소·녹지완충 + 태풍 시나리오 |
| 데이터 기반 설계 | 기상청 ASOS·황사 PM10·재난 정보 baseline |
| 바람길·CFD·건물 높이 | **범위 밖** — 격자 타일 근사 모델 |

### 사용 API · 데이터 (정직 표기)

| 데이터 | 용도 | 현재 상태 |
|--------|------|-----------|
| ASOS 시간자료 | **기상청 API허브** `kma_sfctm2` + 일사 묶음형 (`authKey`) — 기본. 공공데이터포털 `getWthrDataList`는 `HORIZON_ASOS_PROVIDER=portal` 시 | 키 설정 시 라이브, 미설정 시 샘플 폴백 |
| 황사 PM10 (기상자료개방포털) | 대기(미세먼지) 렌즈 baseline | baseline 폴백 → 라이브 연동 예정 |
| 태풍·지진 (API허브) | 재난 시나리오·스트레스 테스트 | 시드 프리셋 → 라이브 연동 예정 |

- 환경 변수: `HORIZON_KMA_API_KEY` = 공공데이터포털 **serviceKey**
- 키 미설정·API 오류 시: 내장 샘플 지역 데이터로 graceful fallback (데모 보장)

### AI 활용 방식
- **통합 AI 코치** (Python FastAPI + LLM): 다축(열섬·대기·재난·농어업) metrics·상충 해석, 개선 제안, 학습 포인트.
- LLM 키 미설정 시: 규칙 기반 폴백 코치.

### 사용자 체험 흐름
1. 지역 선택 (서울/부산 등) → ASOS baseline 기온·일사 로드 (`/designer`).
2. 팔레트에서 타일 배치 → 열섬 히트맵·지표 즉시 갱신.
3. 렌즈 전환(대기·재난·농어업) → **같은 설계**를 다른 환경 시험으로 재평가.
4. 시나리오 레벨(Lv1 열섬 → …) · 스트레스 테스트 · AI 코치 → 재설계.

### 학습되는 기상·환경 원리
- **열섬**: 알베도, 녹지·수면 증발냉각, 불투수면·건물 열 저장
- **대기**: 배출원(도로·건물·공장)·녹지 흡착·완충
- **재난**: 태풍·침수 시나리오 하 방재 시설 역할 (태풍 **생성** 아님)
- **다목표 상충**: 산업·개발 밀도 vs 환경·안전 — 만능 도시는 없음

### 핵심 기능 (현행 구현)

| # | 기능 | 비고 |
|---|------|------|
| 1 | 인터랙티브 격자 설계 + 렌즈 히트맵 | 열섬·대기·재난·농어업 |
| 2 | 기상청 ASOS 기온·일사 baseline | `WeatherDataService` + 폴백 |
| 3 | 다축 평가 + 균형 점수 | `POST /api/designs/evaluate` |
| 4 | 시나리오 레벨 + 설계 이어받기 | `lib/levels.ts` |
| 5 | 스트레스 테스트 타임라인 | 폭염→태풍→대기→수확기 |
| 6 | 통합 AI 코치 | 4축 metrics 프롬프트 |
| 7 | 3D 도시 뷰 | `City3DView` |

메인 진입: **`/designer`** (레거시 `/disaster` → 리다이렉트)

## 5. 시뮬레이션 모델 (교육용 근사)

### 열섬 (격자 표면온도)

```
tileTemp = baseAirTemp
         + solarFactor * (1 - albedo) * solarLoad
         - greenCooling
         - waterCooling
         + neighborBuildingHeat
```

- `baseAirTemp` · `solarLoad`: ASOS `ta` · `icsr` 정규화 (또는 샘플 폴백).
- 타일 예: BUILDING·ROAD(덥게), PARK·TREE·WATER·WETLAND(식게).
- **도시 vs 교외 ΔT 재현이 아님** — 격자 내 타일 간 온도 차·체감용 모델.

### 대기·재난·농어업
- **대기**: 타일별 배출·녹지 흡착 + 확산 1패스 (교육용).
- **재난**: 시나리오 프리셋 + 방재 타일 위험도 (교육용).
- **농어업**: 도시 집계 + 광역 구역 배분 (교육용).

모든 축은 정밀 수치예보·도시계획 SW가 **아님**. UI·문서에 교육용 근사임을 명시.
