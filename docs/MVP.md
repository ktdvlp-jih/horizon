## 4. 선정 아이디어 상세

### 아이디어명
도시 기후 설계자 (Urban Climate Designer)

### 한 줄 설명
빈 도시 격자에 건물·도로·공원·물·나무를 직접 배치하면, 실제 기상 데이터 기반으로 표면 온도 히트맵이 실시간으로 바뀌고, AI 도시 코치가 열섬 완화 설계를 평가·코칭한다.

### 사용자의 핵심 행동 (User Action)
타일을 칠해 도시를 설계 → 열섬 히트맵 변화 관찰 → AI 코치 평가 → 재설계.

### 해결하려는 문제
도시 열섬(Urban Heat Island)을 "설명"이 아니라 "직접 설계 실험"으로 체득하게 한다.

### 사용 API
- 기상자료개방포털 Open API (지점별 기온, 일사량) — 지역 baseline 값.
- 키 미설정 시: 내장 샘플 지역 데이터로 graceful fallback (데모 보장).

### AI 활용 방식
- AI 도시 코치 (평가관 + 가이드 + 학습 리포트 생성기).
- 입력: 타일 구성비, 녹지율, 평균/최고 표면온도, 지역 baseline.
- 출력: 점수, 설계 강·약점, 구체적 개선 제안 3개, 한 줄 학습 포인트.
- LLM 키 미설정 시: 규칙 기반 폴백 코치로 동작.

### 사용자 체험 흐름
1. 지역 선택 (서울/부산/대구 등) → baseline 기온·일사량 로드.
2. 팔레트에서 타일 선택 후 격자에 페인팅 → 변경 즉시(디바운스) 히트맵·지표 갱신.
3. "AI 코치에게 평가받기" → 점수·피드백·개선 제안.
4. 제안 반영해 재설계 → 평균 온도 하락 확인.

### 학습되는 기상 원리
- 알베도/녹지/수면이 표면 온도에 미치는 영향, 열섬 완화 설계 원리, 일사량·기온의 상호작용.

### MVP 핵심 기능 3개
1. **인터랙티브 격자 설계 + 실시간 열섬 히트맵** (프론트 + Spring 시뮬레이션 API).
2. **실제 기상 데이터 기반 시뮬레이션** (지역 baseline → 표면온도 모델, 기상 API 연동 + 폴백).
3. **AI 도시 코치** (Python FastAPI + LLM, 평가·제안·학습 리포트, 규칙 기반 폴백).

### 1분 데모 시나리오
- 0:00 서울 선택 → 전부 건물/도로인 "콘크리트 도시" 기본값, 히트맵 빨강, 평균 34도.
- 0:20 공원·나무·물 타일로 재설계 → 히트맵이 파랑으로 식고 평균 28도로 하락.
- 0:40 "AI 코치" 클릭 → "녹지율 12%→41%로 평균 6도 완화. 남측 도로변에 가로수 추가 제안" 등.
- 0:55 제안 반영 → 점수 상승. 마무리.

### 예상 기술 스택
[TECH_STACK.md](TECH_STACK.md) 그대로: React 19/Vite → Spring Boot 3.5 → Python FastAPI, PostgreSQL, Docker Compose.

### 기존 서비스 대비 차별성
조회형 기상 서비스(Windy/Ventusky/기상청)와 달리 사용자가 환경을 설계하고 결과를 즉시 받는 실험 도구. AI 코치가 학습을 완성.

### 총점
91 / 100.

## 5. 시뮬레이션 모델 (MVP 단순화)

타일별 표면 온도(℃):

```
tileTemp = baseAirTemp
         + solarFactor * (1 - albedo) * solarLoad
         - greenCooling
         - waterCooling
         + neighborBuildingHeat
```

- 타일 속성 (albedo / 냉각계수): BUILDING(0.25/-), ROAD(0.12/-), BARE(0.20/-), PARK(0.30/녹지냉각), TREE(0.25/강한 녹지냉각+그늘), WATER(0.08/증발냉각).
- `solarLoad`는 지역 일사량 정규화 값, `baseAirTemp`는 지역 기온.
- 결과 지표: 평균/최고 표면온도, 녹지율, 불투수면율, baseline 대비 완화 폭(ΔT).
- 정밀 기상 모델이 아니라 "행동→결과" 체감을 위한 교육용 근사 모델임을 UI에 명시.
