## 개발 목적 및 필요성 ※주제 적합성

```
【배경 — 도시 기후 설계자(Urban Climate Planner/Designer)】
기후변화 시대에 도시 계획 초기부터 ‘기후’를 최우선 변수로 두고 도시를 기획·디자인하는 신종 전문 직무가 급부상하고 있다. 기존 도시계획가가 교통·주거·상업 배치에 집중했다면, 도시 기후 설계자는 폭염·집중호우·해수면 상승 등에 도시가 회복력을 갖도록 과학적 데이터(기후·기상 관측, 미기후 분석 등)를 바탕으로 공간을 설계한다. 주요 업무에는 바람길·통풍 고려, 그린·블루 인프라(녹지·수변·스펀지 도시), 재난 회복력 강화(방재·배수 연계) 등이 포함되며, 슈투트가르트(바람길)·싱가포르(정원 속의 도시) 등이 대표 사례다. 국내에서도 기후위기 적응·탄소중립 도시 계획 수요가 늘고 있다.

본 프로젝트는 위 전문 직무를 **그대로 대체하는 설계 소프트웨어가 아니라**, 중·고등학생 등이 **「도시 기후 설계자」 역할을 체험**하며 핵심 개념을 익히는 **교육용 웹 시뮬레이터**이다.

【개발 목적】
기상청 데이터로 도시를 설계하고, 열섬·대기·재난을 검증하는 AI 활용 웹 체험·참여형 시뮬레이터를 개발한다. 사용자는 도시 격자에 건물·녹지·수변·방재·산업 시설 등을 배치하며, 열섬에서 시작해 미세먼지·재난·농어업 렌즈를 단계적으로 확장한다. 여러 환경 문제를 동시에 견디는 균형 잡힌 도시 설계를 학습한다.

【체험과 실제 직무의 연결(교육용 근사)】
· 그린·블루 인프라 → 공원·가로수·수변·습지 타일로 열섬·대기 완화 체험
· 재난 회복력 → 방조제·배수로·대피소·녹지완충으로 태풍·침수 시나리오 대응
· 데이터 기반 설계 → 기상청 ASOS 기온·일사·황사 PM10·재난 정보를 baseline·시나리오로 활용
· (본 프로토타입 범위 밖) 바람길·3D 건물 높이·정밀 CFD 등은 교육용 격자 모델로 단순화

【달성 목표】
· 기상·기후 데이터 기반 문제 해결(설계→결과 확인→재설계) 능력 강화
· AI·데이터 융합 기상·기후 교육 역량 증진
· 기후위기·재난·대기질 등 복합 환경 이슈에 대한 시스템 사고(상충·트레이드오프) 체득
· 실제 신종 직무에 대한 진로·사회적 맥락 이해(역할 체험)

【기존 교육의 한계】
· 영상·문서 중심 단방향 전달 → 변수 조작·결과 확인·재설계의 양방향 루프 부족
· 단일 주제(열섬만, 재난만 등) 위주 → 도시 설계의 다목표 상충을 한 번에 경험하기 어려움
· 전문 직무 소개만 하고 직접 설계·검증을 해보지 못함

【AI·데이터 활용 필요성】
· 기상청 실측·관측 데이터(ASOS 기온·일사, 황사 PM10, 태풍·지진 등)를 baseline·시나리오로 활용
· AI 통합 코치가 다축 평가 결과·약점·상충을 해석·피드백하여 학습 루프 완성
```

## 적용 기술 및 실현 구체성 ※실현 가능성

```
【기술 스택】
· 프론트: React 19, Vite, TypeScript
· 백엔드: Spring Boot 3.5, PostgreSQL
· AI: Python FastAPI, LLM 기반 통합 코치(키 미설정 시 규칙 기반 폴백)
· 배포: Docker Compose, GitHub Actions CI/CD

【핵심 기능 — 프로토타입 이미 구현】
· 인터랙티브 격자 설계 + 렌즈 히트맵(열섬·대기·재난·농어업)
· 시나리오별 레벨(열섬 입문 → 단계적 확장, 이전 설계 이어받기)
· 스트레스 테스트 타임라인, 종합 점수·균형 패널티, 통합 AI 코치, 3D 도시 뷰

【데이터 적용·구현 방식】
· ASOS 기온·일사 → 지역 선택 시 baseline 로드 → 표면온도 근사 모델 입력
· 황사 PM10 → 미세먼지 렌즈 baseline(본선 전 API 연동 완료 예정)
· 태풍·지진 → 시나리오 프리셋 + 격자·타일 기반 위험도 근사(본선 전 API허브 연동 예정)
· 시뮬레이션은 교육용 근사 모델(UI·문서에 명시, 정밀 수치모델 아님)

【AI 활용 단계·방식】
· 설계 완료 후 4축 metrics·상충 분석을 LLM에 전달 → 강점·약점·개선 제안·학습 포인트 생성
· LLM 미설정 시 동일 구조의 규칙 기반 코치로 폴백(데모·시연 보장)
· 개발 도구(Cursor 등) 사용 사실을 제출 서류에 정직 기재(공고 변조·위조 방지)

【일정 내 실현 가능성】
· 예선(~7.3): 개발계획서 제출, 프로토타입 URL 시연 가능
· 본선 사전개발(7.23~8.7): PM10·재난 API 라이브 연동, URL 안정화(고정 터널/호스팅)
· 해커톤(8.21~22): 최종 시연·발표
```

## 체험 요소 및 학습 효과 ※체험·참여형 설계 ※학습 효과

```
【체험 흐름: 시작 → 진행 → 완료】

▶ 시작
· 지역(서울·부산 등)·시나리오(해안도시·내륙 분지·대도시) 선택
· 기본 콘크리트 도시에서 출발, 튜토리얼로 조작법 안내

▶ 진행 (양방향 상호작용)
· 팔레트에서 타일 선택 → 격자 클릭·드래그로 도시 설계(그린·블루·방재 시설 배치)
· 렌즈 탭 전환: 열섬 / 대기(미세먼지) / 재난 / 농어업 — 축별 히트맵·점수 확인
· 레벨 진행: Lv1(열섬) → Lv2(+대기) → Lv3(+재난) → Lv4(+농어업), 이전 설계 이어받기
· 스트레스 테스트: 폭염→태풍→미세먼지→수확기 순 자동 재생, 설계 견고성 점검
· 3D 뷰: 도시 입체 확인, 기후 효과 시각화

▶ 완료
· 종합 점수·균형 패널티 확인
· AI 통합 코치: 다축 강·약점·개선 제안·학습 포인트
· 코치 반영 후 재설계 → 점수 변화 확인

【학습 성과】
· 녹지·수면(그린·블루)이 열섬·대기에 동시에 기여하는 원리 설명 가능
· 방재시설(방조제·배수로·대피소) 우선순위를 스스로 판단
· 산업(공장) 배치와 대기·개발 상충을 설명(완충 녹지·공존 관점)
· 한 축만 잘하는 설계의 한계(균형 패널티)를 이해하고 다축 균형 관점으로 평가
· 실제 「도시 기후 설계자」 직무가 다루는 문제(열섬·대기·재난)와의 연관성 설명 가능
```

## 창의성 및 차별성 ※아이디어 독창성

```
【차별화 포인트】
· 실제 신종 직무 「도시 기후 설계자」를 **역할 체험**으로 재현 — 조회형 기상앱·단일 개념 교육과 구분
· ‘하나의 도시 설계’를 열섬·대기·재난·농어업 렌즈로 **단계적·다각 검증**하는 다중 렌즈 컨셉
· 시나리오별 고정 가중치 + 균형 패널티 → 한 축만 잘하는 ‘만능 도시’ 착시 방지
· 레벨 누적 학습(이전 설계 이어받기) → 변경점이 결과에 누적 반영되는 체감
· 스트레스 테스트 타임라인 → 1년간 기후·재난 충격을 순차 점검

【실제 직무·도시 사례와의 관계(정직 표기)】
· 슈투트가르트(바람길)·싱가포르(그린 도시) 등은 **학습·발표 맥락**으로 인용. 본 서비스는 이를 **정밀 재현하지 않음**
· 바람길·건물 높이·CFD는 프로토타입 범위 밖 → 격자 타일·기상청 baseline 기반 **교육용 근사**
· 태풍 **생성·발달** 시뮬이 아니라, **재난 시나리오 하 도시 방재 설계** 평가

【공고 『대회 주제 예시』12건과의 관계 — 표절 회피】
본 서비스는 아래 예시를 ‘그대로 구현’하지 않음. 핵심은 「도시 격자 설계 + 다축 렌즈 + AI 코치」.

· 예시 1~5(날씨 탐험·구름·비·바람·계절): 해당 없음 — 초등 단일 개념·퀴즈형 아님
· 예시 6(기후변화 CO₂·해수면 슬라이더): 해당 없음 — 슬라이더로 지구온난화 설명하지 않음
· 예시 7(기상특보 판단): 해당 없음 — 데이터로 특보를 ‘판정’하지 않음
· 예시 8(태풍 형성·발달): ⚠ 표현 주의 — 태풍을 ‘만들지’ 않음. 태풍 시나리오 하에 도시 방재 설계 위험도 평가
· 예시 9(예보관 의사결정): 해당 없음 — 날씨 예보하지 않음
· 예시 10·11(AI 데이터 분석·NWP): 해당 없음 — AI는 설계 코치 역할
· 예시 12(기후위기 체감): ⚠ 표현 주의 — 미래 vs 현재 기온 비교 UI 아님. ‘내 도시’가 충격을 버티는지 검증

【Horizon 금지 아이디어】태풍 시뮬레이터·예보관·바람·기압 지도·단순 대시보드 → 해당 없음

【기존 대비】
· 기상청·Windy 등 조회형 서비스: 사용자가 환경을 설계·변경하지 못함
· 단일 축 시뮬레이터: 열섬만, 재난만 등 분리 → 시스템 사고 학습 한계
· 본 서비스: 설계-평가-코치-재설계 루프가 하나의 URL에서 완결, AI가 다축 상충 해석

【새로운 인터랙션·접근】
· 렌즈 전환으로 동일 격자를 4가지 환경 시험으로 재해석
· 광역 농어업 레이어(도시 외곽 구역 배분)로 좁은 격자 한계 극복
· AI 통합 코치가 다축 metrics를 한 번에 해석(단일 축 코치 대비 확장)
```

## 부록 — 도시 기후 설계자(전문 직무) 요약 ※발표·Q&A 참고

```
【정의】
기후변화(폭염·집중호우·해수면 상승 등)에 도시가 회복력을 갖도록, 과학적 데이터를 바탕으로 도시 공간을 기획·디자인하는 신종 전문 직무. 기존 도시계획가의 교통·주거·상업 중심 설계를 넘어 ‘기후’를 최우선 변수로 둔다.

【주요 업무】
· 바람길(Wind Corridor): 외곽 찬 공기 유입·통풍을 고려한 건물·도로 배치
· 그린·블루 인프라: 녹지·수변·습지로 열섬 완화·홍수 흡수(스펀지 도시)
· 재난 회복력: 집중호우·해수면 상승 대비 방재·배수 연계 계획

【왜 필요한가】
도시 인구·온실가스·기후 피해가 집중되는 시대에, 콘크리트 중심 확장만으로는 폭염·홍수 피해를 감당하기 어렵다. 국내 기후위기 적응·탄소중립 도시 계획 수요도 증가 중.

【본 프로젝트와의 관계】
웹 체험 「도시 기후 설계자」는 위 직무를 **역할 체험·입문 교육**으로 소개한다. 사용자는 격자 설계·렌즈 평가·AI 코치를 통해 **그린·블루·방재·데이터 기반 설계**를 체험하되, 정밀 도시계획·바람 시뮬레이션을 대체하지 않는다.
```
